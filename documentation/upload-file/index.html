Upload File

A simple form to upload AIS documents (AIP, amendments, supplements, AICs) so the AIS team can publish them.

Step-by-step (what to do)

  1. Select a Category: Choose one of:
    • AIP — full AIP parts and chapters
    • AMDT — amendment to existing AIP material
    • SUP — supplement (choose Bravo/Charlie series)
    • AIC — aeronautical information circular
  3. If you chose SUP: pick a Path Name (Bravo or Charlie).
  4. If you chose AIP: pick a Part (Part 1 = GEN, Part 2 = ENR, Part 3 = AD) then choose the Chapter/Number. Some Part 3 (AD) selections show extra sub‑choices (e.g., INTERNATIONAL, MILITARY).
  5. Subsection: Short identifier or subsection name (required).
  6. Title: Full title of the document. (For AMDT, include the effective date in the title if needed.)
  7. Batch Date Reference (optional): Pick a CURRENT or UPCOMING batch to auto-fill Publication and Effectivity dates, or leave blank to enter those dates manually.
  8. Publication Date & Effectivity Date: Enter when the document is published and when it becomes effective (required).
  9. Expiration Type: Choose:
    • Self-Expiratory — you must enter an End Date.
    • PERM — permanent; no end date.
      The form will show/hide the End Date field based on this choice.
  11. Upload PDF: Attach the document as a PDF only (.pdf). This is required.
  12. Submit Document: Click the Submit button; a success message appears if upload completes.

Quick tips

  • Use a Batch Date to avoid manual date entry errors.
  • For AIP uploads, confirm you picked the right Part and Chapter to keep documents organized.
  • Double-check expiration dates if the document should expire (Self-Expiratory).
  • Ensure your file is a PDF and not password-protected.

AIS Document Upload: Best Practices & Troubleshooting

A. Category Selection

  • Always select the correct Category:
    • AIP, AMDT, SUP, or AIC.
  • Double-check the category before submitting.
    • Example: SUP for Supplements, AIC for Circulars.

B. Title Field

  • The Title must clearly identify the document and its type.
  • For AIC:
    • Always include "AIC" in the title.
    • Example: AIC 002/26 AIRAC SYSTEM/AMENDMENTS TO AIP PHILIPPINES FOR YEAR 2025
  • For SUP:
    • Include "SUP" in the title.
    • Example: 001/26 CHECKLIST OF AIP SUPPLEMENTS
  • For AMDT:
    • Example: AIRAC AIP AMDT 001/26 22 JAN 26
  • For AIP:
    • Example: GEN 1.2 ENTRY, TRANSIT AND DEPARTURE OF AIRCRAFT | ENR 1.1 GENERAL RULES AND PROCEDURES | AD 1.1 AERODROME/HELIPORT AVAILABILITY AND CONDITIONS OF USE | AD RPSP BOHOL-PANGLAO INTERNATIONAL AIRPORT | AD RPVB BACOLOD PRINCIPAL AIRPORT (CLASS 1) | AD RPVS ANTIQUE PRINCIPAL AIRPORT (CLASS 2) | AD RPLY ALABAT COMMUNITY AIRPORT | AD RPML CAGAYAN DE ORO (MILITARY)

C. Subsection Field

  • Use a unique and descriptive subsection name.
  • Avoid generic names like "Update" or "Notice".

D. Category-Specific Fields

  • For SUP: Select the correct path (Bravo/Charlie).
  • For AIP: Select the correct Part and Chapter.
  • For AD Part 3, Chapter 2: Select the correct Sub Chapter (e.g., INTERNATIONAL, PRINCIPAL - CLASS 1).

E. Category/Chapter Consistency

  • Ensure the category, chapter, and sub chapter match the document’s content and intended audience.


Common Issues & Solutions

Issue 1: File Not Displaying for SUP (or AIC) Due to Duplicate Title

  • Cause: Two files with the same title, one in SUP and one in AIC, but the AIC file does not have "AIC" in the title.
  • Solution:
    • Always include the category (e.g., "AIC", "SUP") in the title.
    • If a file is missing, check for duplicate titles across categories.
    • Rename the title to include the correct category and re-upload if needed.

Issue 2: Incorrect Category or Sub Chapter (e.g., RPVM assigned to Principal Class 1 instead of International)

  • Cause: File uploaded to the wrong sub chapter/category.
  • Solution:
    • Archive or delete the incorrectly categorized file.
    • Re-upload the file under the correct category/sub chapter.
    • Ensure the title and sub chapter match the document’s actual content.

Issue 3: File with Same Title Still Showing (Old File Not Expired)

  • Cause: System displays the most recent non-expired file with a given title.
  • Solution:
    • Archive or set the old file to expired if it’s in the wrong category.
    • If archiving is not possible, delete the old file before re-uploading.
    • Always check the expiration date and status of existing files before uploading a new one with the same title.

Issue 4: Overwriting or Replacing Files

  • Best Practice:
    • Never upload a new file with the same title and category unless you intend to replace the old one.
    • If a correction is needed, archive or expire the old file first.

Issue 5: Incorrect Date Fields

  • Solution:
    • Ensure publication and effectivity dates are correct and in the future.
    • For self-expiratory files, the expiration date must be after the effectivity date.


Preventive Measures

  • Always double-check all fields before submitting.
  • Use unique, descriptive titles that include the category.
  • Archive or expire old/incorrect files before uploading corrections.
  • For corrections, do not just re-upload—ensure the old file is not active.
  • If unsure, consult this guide or contact the system administrator.

Additional Possible Issues & Solutions

  • Uploading a file with a typo in the title:
    • Solution: Delete or archive the file, correct the title, and re-upload.
  • Uploading to the wrong effective date:
    • Solution: Archive/delete the file and re-upload with the correct date.
  • Uploading a file with the wrong PDF:
    • Solution: Archive/delete the file and re-upload with the correct document.


Quick Checklist Before Upload

  •  Correct category selected
  •  Title includes category and is unique
  •  Subsection and sub chapter are accurate
  •  Dates are correct and valid
  •  No duplicate active files with the same title/category
